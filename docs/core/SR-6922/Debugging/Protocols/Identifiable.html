  • readableName Default implementation

    A readable name for the error’s Type. This is usually similar to the Type name of the error with spaces added. This will normally be printed proceeding the error’s reason.

    Note

    For example, an error named FooError will have the readableName "Foo Error".

    Default Implementation

    Default implementation of readable name that expands SomeModule.MyType.Error => My Type Error

  • reason

    The reason for the error. Typical implementations will switch over self and return a friendly String describing the error.

    Note

    It is most convenient that self be a Swift.Error.

    Here is one way to do this:

    switch self {
case someError:
   return "A `String` describing what went wrong including the actual error: `Error.someError`."
